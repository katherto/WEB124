How do HTML id and class attributes connect to CSS selectors?

1. <p id="xyz">Item number xyz</p> connects to what selector in the css?

Connects to the Id selector denoted by # in CSS (as in #xyz)

2. How many html elements may have the same id="xyz" attribute value on a given page?

Only one

3. <p class="holidayItem">Seasonal item for sale.</p> connects to what selector in the css?

Connects to the Class selector denoted by . in CSS (as in .holidayItem)

4. How many html elements may have the same class="holidayItem" attribute value on a given page?

As many as deemed necessary

Essential JavaScript best practices: Explain the pros and cons of...

1. inline vs. embedded vs. external JavaScript using examples.

Pro: Inline or Embedded JavaScript can provide for a way to easily test some idea without need of creating a new file and properly linking it within the HTML code. External JavaScript is favored as it keeps the behavioral aspects of a webpage separate from the content and structure.

Con: Using inline or embedded JavaScript can make it harder to update specific parts of a webpage and can make it harder to read

Code Example:
let today = new Date();
let hourNow = today.getHours();
let greeting;

if (hourNow > 18) {
greeting = 'Good evening!';
} else if (hourNow > 12) {
greeting = 'Good afternoon!';
} else if (hourNow > 0) {
greeting = 'Good morning!';
} else {
greeting = 'Welcome!';
}

By including all this code within a script tag in the middle of HTML code, if for some reason the JavaScript for the page breaks due to an update someone might not think to browse the HTML document itself and find these specific lines to correct.

2. indenting your code.

Pro: Makes the resulting page of code more readable for others to follow along and update as necessary

Con: It may take more time and if indenting is inconsistent will result in code that's harder to read

Code Example:
if (x > 1) {
    return false;
}

vs.

if (x > 1) { return false}

3. inserting code comments

Pro: Code comments are essential for documenting the work you're doing while programming. It'll help others who update the code with important information that help explains logic or what the code is programmed to do.

Con: Code comments also take more time than straight programming without them. If comments are too general or vague there is risk of having poorly documented code that doesn't explain itself well.

Code Example:
// each time an h2 element is clicked random hex values are chosen and set as a gradient for the background
document.body.style.backgroundImage = "linear-gradient(to bottom, #" + Math.floor(Math.random()*16777215).toString(16) + ", #" + Math.floor(Math.random()*16777215).toString(16);

Without the comment someone following up on this project may wonder why random numbers are being generated and what it has to do with styling with JavaScript

Java vs. JavaScript

  • What is the general purpose for each language?
    Java is generally used to create mobile and desktop apps, web apps, games, etc. JavaScript is used to control the behavior of a webpage and certain frameworks of JavaScript can be used to create more specific things like mobile apps.
  • Strict vs. loose variable typing - what does this mean as it relates to Java and JavaScript?
    Loose variable typing allows you to assign a name/value pair without first declaring the variable itself, whereas strict explicity expects a variable definition before assigning it a value. In JavaScript, to use strict typing an instruction "use_strict"; is set at the beginning of the JavaScript document.
  • Where is the source code for each language compiled or interpreted?
    For Java, a Java compiler reduces source code to bytecode, then is run on a Java Virtual Machine. In contrast, the major browsers running JavaScript have an interpreter to translate the code into something the computer can follow.
