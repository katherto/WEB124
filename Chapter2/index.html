Review Questions

1. When does the JavaScript in this project run?

The JavaScript runs once the browser renders the page and encounters the script tag. In this case the whole page has displayed and did show "Johnson County Community College" in the six paragraphs before encountering the script tag, loading, and executing the code which then changed each paragraph as we instructed.

2. Why can you + text with a number but not * text with a number in JavaScript?

In JavaScript you can use + to either perform math or concatenate text. However, you can't perform math on text with * as it's expecting valid number inputs so it can give a number output, which, is why the result was NaN for paragraph five.

3. What is the difference between the textContent and innerHTML properties and why was it more appropriate to use textContent for this exercise?

innerHTML allows us to type HTML code as part of the string when we modify the innerHTML property of an element in JavaScript. The HTML code will be styled and reflected as it should. With textContent, any HTML code is actually shown as part of the text when we modify the textContent property in JS. No styling will be reflected when changing the content of an element with this property. Best practice usually is to only edit textContent with Javascript for an existing element, and add a new element with Javascript instead of using innerHTML. Since we already created the paragraph elements with text beforehand, we worked with the textContent property in JS.

4. What is "automatic semicolon insertion" in JavaScript?

If a statement is not explicitly terminated with a semicolon, in certain situations a semicolon will be automatically inserted by JavaScript engine. This has the potential to alter the behavior of our JavaScript code.

5. When is it OK to break a statement onto two lines in Javascript without using special characters?

Typically this happens if there's a long line of logical operators in an expression. The extra returns assist our code in being more readable while the return character itself in the code influences how ASI works.

6. How are tabs and spaces used and treated in Javascript?

Spaces and tabs are generally ignored in JavaScript. You can include spaces and tabs at any point on a line of code and it would be seen and executed the same. These are another means of helping our code be more readable.
