Comparison Operators: Create and complete the following Table

Operators Meaning Example and Return Value Explanation of Example
== Equals a == b (returns false) The two operands, a and b, are not equal so this expression will return a false value.
=== Strict Equals '10' === 10 (returns false) Strict equals compares data types and prevents JavaScript from performing type coercion on the string 10. Since the string remains a string and is compared to a number value this expression returns a false value.
!= Not Equals 20 != 10 (returns true) Not equals returns the opposite of what the equals operator would. Since 20 does not equal 10, and equals would normally return false, not equals will return a true value.
!== Strict Not Equals '50' !== '50' (returns false) Strict not equals returns the opposite of what the strict equals operator would. Since the two strings '50' equal each other and match data types, strict equals would return true, but strict not equals in this case will return a false value.
> Greater Than 43 > 32 (returns true) The operands 43 and 32 are compared and since in this case 43 is greater than 32 a true value will be returned for this expression.
>= Greater Than or Equal to 43 >= 43 (returns true) In this case even though the two operands are compared and neither is greater than the other, since they are equal and the operator includes that case a true value will be returned.
< Less Than 43 < 32 (returns false) The operands 43 and 32 are compared and since in this case 43 is greater than 32 a false value will be returned for this expression.
<= Less Than or Equal to 32 <= 32 (returns true) In this case even though the two operands are compared and neither is less than the other, since they are equal and the operator includes that case a true value will be returned.
&& Logical And (43 > 32) && (32 < 43) (returns true) The left side of this expression is evaluated first, and since 43 is greater than 32 it evaluates true. However, with Logical And both sides of the expression must be true for it to return true. Thus, the right side is evaluated and since 32 is less than 43 and evaluates true, the whole expression returns a true value.
|| Logical Or (43 < 32) || (43 > 32) (returns true) The left side of this expression is evaluated first, and since 43 is greater than 32 it evaluates false. However, with Logical Or only one side has to evaluate as true in order to return a true value. Thus, the right side is evaluated and since 43 is greater than 32, the whole expression returns a true value.
! Logical Not !true (returns false) Not simply returns the opposite of the logical expression. In this case the opposite of true is false, and the expression will return a false value.
